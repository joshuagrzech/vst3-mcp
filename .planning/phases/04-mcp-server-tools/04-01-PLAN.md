<task type="auto">
  <name>Task 1: Add six MCP parameter tools to server.rs</name>
  <files>src/server.rs</files>
  <action>
Add six new MCP tools to the AudioHost impl block (after load_preset, before the closing brace). Follow the existing #[tool] pattern from scan_plugins and load_plugin.

**Tool 1: get_plugin_info**
- No request parameters (empty struct or no Parameters wrapper)
- Lock self.plugin_info, extract classId (uid), name, vendor
- Return JSON: {"classId": "...", "name": "...", "vendor": "..."}
- Error if no plugin loaded

**Tool 2: list_params**
- No request parameters
- Lock self.plugin, call get_parameter_count()
- Loop through parameters (0..count):
  - Call get_parameter_info(i)
  - Filter: ONLY include if info.is_writable() && !info.is_hidden()
  - For each included param: get_parameter(id), get_parameter_display(id)
  - Build JSON object: {"id": id, "name": title, "value": value, "display": display}
- Return JSON array: {"parameters": [...], "count": N}
- Error if no plugin loaded

**Tool 3: get_param**
- Request struct: GetParamRequest { id: u32 }
- Lock self.plugin, call get_parameter(id), get_parameter_display(id)
- Return JSON: {"id": id, "value": value, "display": display}
- Error if no plugin loaded

**Tool 4: set_param**
- Request struct: SetParamRequest { id: u32, value: f64 }
- Validate: value must be in range [0.0, 1.0], return error otherwise
- Lock self.plugin (mutable), call queue_parameter_change(id, value)
- Get display string for confirmation
- Return JSON: {"status": "queued", "id": id, "value": value, "display": display}
- Note in tool description: "Changes are queued and applied in next process_audio call"

**Tool 5: batch_set**
- Request struct: BatchSetRequest { changes: Vec<ParamChange> }
- ParamChange struct: { id: u32, value: f64 }
- Validate ALL changes first (all values in [0.0, 1.0])
- If any invalid, return error with first invalid change details
- Lock self.plugin (mutable), queue all changes via loop
- Return JSON: {"status": "queued", "changes_queued": N}
- Atomic behavior: validate all before queuing any

**Tool descriptions:**
- Use the #[tool(description = "...")] attribute for each tool
- Description should explain what it returns and any prerequisites (e.g., "Call load_plugin first")

**Error handling:**
- All tools return Result<String, String>
- Lock errors: map_err with "Lock error: {}"
- No plugin loaded: "No plugin loaded. Call load_plugin first."
- Invalid parameter values: specific error with actual value shown

**Request struct placement:**
- Add all request structs (GetParamRequest, SetParamRequest, BatchSetRequest, ParamChange) at the top of server.rs after existing request structs (after LoadPresetRequest)
- Derive Debug, serde::Deserialize, schemars::JsonSchema for all
- Use #[schemars(description = "...")] for field documentation

**Imports:**
- No new imports needed (serde_json::json! already used, all plugin methods exist)
