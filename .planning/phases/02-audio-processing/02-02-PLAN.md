Add the following test functions to `tests/audio_processing.rs`. All tests use the plugin loading helper from Task 1. Tests that require a real plugin should gracefully skip if none is available.

**Test 1 -- Success Criterion #1: WAV file through effect plugin produces valid output**
`fn test_process_wav_through_plugin()`
- Generate a 2-second stereo 44100Hz WAV test file
- Load plugin, decode test file, render_offline, write output WAV
- Assert: output file exists, output file is valid WAV (can be decoded back), output has same channel count as input, output frame count >= input frame count (may be longer due to tail), output sample rate matches input
- Assert: output is not all zeros (plugin did something or passed through)

**Test 2 -- Success Criterion #2: Transparent/bypass produces near-identical output**
`fn test_bypass_produces_near_identical()`
- Generate a 1-second stereo test file
- Load plugin. If the plugin has a "bypass" or "dry/wet" parameter, set it to bypass. If not, this test validates that processing with default settings at least preserves the signal structure (RMS of output within reasonable range of input RMS, e.g., within 20dB).
- For a true passthrough test without relying on plugin bypass: Process silence through the plugin and verify output is near-silence (max absolute sample < 1e-6 after a few blocks of warmup). This confirms no audio corruption from the pipeline itself.
- Actually, the most reliable test: generate a known signal, process through the plugin, then verify the output's RMS is within a reasonable range of the input's RMS. An effect plugin should not add 40dB of gain or reduce to silence unless specifically configured.

**Test 3 -- Success Criterion #3: Stereo channel preservation**
`fn test_stereo_channels_preserved()`
- Generate stereo WAV with distinct L (440Hz) and R (880Hz) content
- Process through plugin
- Deinterleave output
- Cross-correlate: output_left with input_left should be higher than output_left with input_right
- Cross-correlate: output_right with input_right should be higher than output_right with input_left
- This proves no channel swap occurred
- Also verify output has 2 channels (no mono collapse)

**Test 4 -- Success Criterion #4: Sample rate preservation**
`fn test_sample_rate_preserved()`
- Generate test files at two different sample rates: 44100 and 48000
- Process each through the plugin
- Decode output files
- Assert: each output's sample rate matches its input's sample rate

**Test 5 -- Success Criterion #5: No buffer boundary artifacts**
`fn test_no_buffer_boundary_artifacts()`
- Generate a long-ish stereo WAV (e.g., 3 seconds at 44100Hz = 132300 frames, which crosses many 4096-sample block boundaries -- 32+ blocks)
- Process through plugin
- Check for discontinuities at block boundaries: iterate through output samples looking for sudden jumps. Compute the difference between consecutive samples; if processing a smooth sine wave, the max frame-to-frame delta should be bounded (a 440Hz sine at 44100Hz has max delta of about 2*pi*440/44100 ~= 0.0627 per sample). Allow a generous threshold (e.g., 0.5) to account for plugin effects, but flag if any delta exceeds 1.0 which would indicate a click/pop.
- This is a heuristic test. If the plugin applies heavy effects (distortion, compression), the thresholds need to be more relaxed. Use a reasonable threshold and document the assumption.

**General patterns:**
- Each test creates a temp directory, generates test audio, processes it, reads back the output, and asserts criteria
- Use `#[test]` attribute (standard integration test)
- If plugin loading fails, print a message and return early (don't panic) -- treat as "skipped"
- Clean up temp files after each test
