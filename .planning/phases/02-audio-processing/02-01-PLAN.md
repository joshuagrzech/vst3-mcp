**Fix 1: Correct bus count in ProcessData**

During `setup()`, after `activate_default_buses()`, query and store:

let num_input_audio_buses = unsafe {
    self.component.getBusCount(kAudio as i32, kInput as i32)

let num_output_audio_buses = unsafe {
    self.component.getBusCount(kAudio as i32, kOutput as i32)

context.projectTimeSamples = self.sample_position;

Set `process_data.processContext = &mut context;` and advance `self.sample_position += num_samples as i64` after each call.

Add `current_sample_rate: f64` field, set during setup().

**Denormal flushing in process.rs:**

Add FTZ+DAZ flag management around the process loop in `render_offline()`. Use inline `_mm_setcsr` (no new dependency), x86_64 only (the target platform per PROJECT.md).

Add helper functions at the top of process.rs:
```rust
#[cfg(target_arch = "x86_64")]
unsafe fn set_ftz_daz() -> u32 {
    let old = std::arch::x86_64::_mm_getcsr();
    // FTZ = bit 15, DAZ = bit 6
    std::arch::x86_64::_mm_setcsr(old | (1 << 15) | (1 << 6));
    old
}

#[cfg(target_arch = "x86_64")]
unsafe fn restore_mxcsr(old: u32) {
    std::arch::x86_64::_mm_setcsr(old);
}
```

In `render_offline()`, wrap the processing section (steps 5 + 6, the two while loops) with save/restore:
```rust
#[cfg(target_arch = "x86_64")]
let old_mxcsr = unsafe { set_ftz_daz() };

// ... existing process loops ...

#[cfg(target_arch = "x86_64")]
unsafe { restore_mxcsr(old_mxcsr); }
```

Use a scope guard pattern or ensure restore happens even on error. The simplest correct approach: wrap the processing in a closure or use a struct with Drop. Given this is offline (not real-time), a simple approach with `?` early returns is fine -- but make sure restore_mxcsr is called before the return. Best approach: use `defer`-style pattern or explicitly handle:

```rust
let result = (|| -> Result<()> {
    // process loops here
    Ok(())
})();

#[cfg(target_arch = "x86_64")]
unsafe { restore_mxcsr(old_mxcsr); }

result?;
```
