<objective>
Harden the VST3 hosting layer for crash-safe scanning and correct teardown ordering.

Purpose: The existing hosting code works but has two structural gaps that the research identified: (1) scanning runs in-process so a buggy plugin can crash the host, and (2) the Drop implementation relies on implicit ordering rather than explicit COM pointer release sequencing. This plan closes both gaps.

Output: A standalone scanner binary, an out-of-process scan coordinator, a hardened PluginInstance::Drop with Option::take(), and Arc<VstModule> lifetime enforcement.
</objective>

<task type="auto">
  <name>Task 2: Harden teardown ordering and enforce module lifetime</name>
  <files>
    src/hosting/plugin.rs

  <action>
In `src/hosting/plugin.rs`, make these changes to `PluginInstance`:

1. **Add `Arc<VstModule>` field** to enforce module lifetime:
   - Add `use std::sync::Arc;` import
   - Add field `_module: Arc<VstModule>` to the struct (after `param_changes`)
   - Update `from_factory()` signature to accept `module: Arc<VstModule>` parameter (add it as the first parameter, before `factory`)
   - Store `module` in the struct
   - The factory can be obtained from `module.factory()` inside the function
   - This guarantees the module (and its dlopen handle) outlives the plugin instance

2. **Harden the `Drop` impl** with explicit `Option::take()`:
   - Change connection point disconnect to use `.take()`:
     ```rust
     if let (Some(ccp), Some(kcp)) = (
         self._comp_connection.take(),
         self._ctrl_connection.take(),
     ) {
         unsafe {
             let _ = ccp.disconnect(kcp.as_ptr());
             let _ = kcp.disconnect(ccp.as_ptr());
         }
         // ccp and kcp dropped here, releasing COM references BEFORE terminate
     }
     ```
   - Change controller termination to use `.take()`:
     ```rust
     if let Some(ctrl) = self.controller.take() {
         let comp_as_ctrl: Option<ComPtr<IEditController>> = self.component.cast();
         let is_same = comp_as_ctrl.as_ref().is_some_and(|c| {
             std::ptr::eq(c.as_ptr(), ctrl.as_ptr())
         });
         if !is_same {
             unsafe { let _ = ctrl.terminate(); }
         }
         // ctrl dropped here, releasing COM reference BEFORE component terminate
     }
     ```
   - The key difference: `.take()` moves the value OUT of the Option, so it is dropped at the end of the `if let` block, not when the struct is dropped. This ensures COM Release() happens in the correct order relative to terminate() calls.

In `src/hosting/module.rs`, harden `VstModule::Drop`:

1. **Use `ManuallyDrop` for the factory** to make drop order explicit:
   - Add `use std::mem::ManuallyDrop;`
   - Change `factory` field to `factory: ManuallyDrop<ComPtr<IPluginFactory>>`
   - In the constructor, wrap: `ManuallyDrop::new(factory)`
   - Update `factory()` method to return `&ComPtr<IPluginFactory>` via `&*self.factory`
   - In `Drop`, explicitly drop factory before calling ExitDll:
     ```rust
     fn drop(&mut self) {
         // Drop factory COM pointer FIRST (releases reference to plugin code)
         unsafe { ManuallyDrop::drop(&mut self.factory); }
         // THEN call ExitDll
         unsafe {
             if let Ok(exit_dll) = self._library.get::<ExitDllFn>(b"ExitDll\0") {
                 exit_dll();
             }
         }
         // Library handle dropped last when struct is dropped
     }
     ```

2. Update all call sites that create `PluginInstance::from_factory()` to pass `Arc<VstModule>`. Check `src/server.rs` and any other callers -- they need to wrap `VstModule` in `Arc::new()` and pass it to `from_factory()`.
