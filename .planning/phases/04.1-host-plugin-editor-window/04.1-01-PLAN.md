Create src/gui/window.rs implementing EditorWindow struct and open_editor_window() entry point:

EditorWindow fields:
- window: winit::window::Window
- plug_view: ComPtr&lt;IPlugView&gt;
- plugin_window_id: Option&lt;u32&gt; (for XEmbed in Plan 02)

EditorWindow::new() implementation (Pattern 1 from research):
1. Lock plugin mutex to access controller
2. Call controller.createView(b"editor\0") to get IPlugView
3. Call plug_view.isPlatformTypeSupported("X11EmbedWindowID") and check result is kResultOk
4. Call plug_view.getSize(&amp;mut width, &amp;mut height) to query preferred size (default 800x600 if fails)
5. Create winit window with WindowBuilder::new().with_title("Plugin Editor").with_inner_size(PhysicalSize::new(width, height)).build()
6. Extract X11 Window ID from window.raw_window_handle() (match RawWindowHandle::Xlib or Xcb, error if not X11)
7. Create IPlugFrame (from plugframe.rs) and call plug_view.setFrame(plug_frame_ptr)
8. Call plug_view.attached(parent_window_id as *mut c_void, "X11EmbedWindowID") and check result
9. Log success with window ID and dimensions

EditorWindow::Drop implementation:
1. Call plug_view.removed() (CRITICAL: must happen before window drops)
2. Call plug_view.setFrame(std::ptr::null_mut()) to clear frame reference
3. Log "Editor window dropped"

open_editor_window() function:
1. Takes Arc&lt;Mutex&lt;Option&lt;PluginInstance&gt;&gt;&gt;
2. Creates winit::event_loop::EventLoop::new()
3. Creates EditorWindow::new(&amp;event_loop, plugin)
4. Runs event_loop.run() with simple event handler (log events, exit on CloseRequested)
5. Returns Ok("editor closed") when loop exits

<task type="auto">
  <name>Task 3: Add open_editor MCP tool with dedicated GUI thread</name>
  <files>
src/server.rs
  </files>
  <action>
Add open_editor MCP tool method to AudioHost impl:

Signature: fn open_editor(&amp;self) -&gt; Result&lt;String, String&gt;

Implementation (Pattern 4 threading model from research):
1. Check self.plugin.lock().unwrap().is_none() and return Err("No plugin loaded. Call load_plugin first.") if no plugin
2. Clone Arc&lt;Mutex&lt;Option&lt;PluginInstance&gt;&gt;&gt; for GUI thread: let plugin_arc = Arc::clone(&amp;self.plugin);
3. Create oneshot channel: let (tx, rx) = tokio::sync::oneshot::channel();
4. Spawn std::thread::spawn (NOT tokio::spawn, winit event loop is blocking):
   - Call gui::open_editor_window(plugin_arc)
   - On success: info!("Editor closed"), tx.send(Ok(()))
   - On error: error!("Editor failed: {}", e), tx.send(Err(e))
5. Block until window closes: tokio::task::block_in_place(|| rx.blocking_recv().map_err(|_| "GUI thread panicked")?)
6. Return Ok("Editor closed")

Add #[tool(description = "Open the plugin's graphical editor window (blocks until window is closed)")] attribute.

Add use gui::open_editor_window; at top of server.rs.

Note: This tool BLOCKS the MCP call until the window closes. This is intentional per research Open Question 4 -- simpler for Phase 04.1 where AI opens editor, user interacts, then closes editor before AI continues.
